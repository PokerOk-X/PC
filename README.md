## Вспомогательный скрипт pokerok.py

Небольшая утилита на Python для проверки установочных файлов клиента PokerOk: `pokerok.dmg` (macOS) и `pokerok.msi` (Windows). Скрипт выводит базовую информацию о файлах (размер, даты), считает SHA-256 и, если доступно, подтягивает системные данные: на macOS — `hdiutil imageinfo`, на Windows — цифровую подпись через `Get-AuthenticodeSignature`.

**Возможности**:

- Проверка наличия `pokerok.dmg` и `pokerok.msi`
- Размер файла и временные метки (created/modified)
- SHA-256 для сверки целостности после загрузки на компьютер
- Подсказка типа файла (если доступна утилита `file`)
- macOS: информация об образе DMG через `hdiutil`
- Windows: проверка подписи MSI через PowerShell / pwsh (Authenticode)
- Вывод отчёта в JSON (удобно для логов/CI)

**Требования**:

- Python 3.8+ (рекомендуется 3.10+)
- ОС: Windows или macOS (Linux тоже запустит базовую часть, но без DMG/MSI-специфики)
- Дополнительно (необязательно):
  - macOS: `hdiutil` (обычно есть по умолчанию)
  - Windows: `powershell` или `pwsh` (обычно есть) для проверки подписи
  - Unix/macOS/WSL: `file` для определения типа файла

**Установка**:

Скрипт не требует зависимостей.

1. Скачайте `pokerok.py` в папку проекта
2. Положите рядом файлы `pokerok.dmg` и/или `pokerok.msi`  
   (или отредактируйте список `FILES` в коде и укажите свои пути)

**Быстрый старт**:

1) Запуск из папки с файлами

```bash
python3 pokerok.py
````

2) Windows (PowerShell)

```powershell
python .\pokerok.py
```

3) Пример вывода (фрагмент)

```json
{
  "platform": {
    "system": "Windows",
    "release": "10",
    "machine": "AMD64",
    "python": "3.11.6"
  },
  "files": [
    {
      "path": "C:\\Users\\you\\Downloads\\pokerok.msi",
      "exists": true,
      "size_bytes": 123456789,
      "modified": "2025-12-29 12:34:56",
      "created": "2025-12-29 12:33:10",
      "sha256": "…",
      "type": "msi",
      "authenticode": {
        "returncode": 0,
        "stdout": "{\"Status\":\"Valid\",...}"
      }
    }
  ]
}
```

**Как интерпретировать результаты**:

* `sha256` — используйте для сравнения скачанного файла с эталонным (если есть официальный хэш) или для контроля “файл не изменился”.
* `authenticode` (Windows) — если `Status: Valid`, у MSI есть валидная подпись (это хороший сигнал).
* `hdiutil_imageinfo` (macOS) — помогает понять, что DMG читается и корректно распознаётся системой.

**Ограничения**:

* Скрипт не скачивает файлы и не устанавливает клиент — он только анализирует уже загруженные установщики.
* Проверка подписи MSI выполняется только на Windows при наличии PowerShell.
* Детальная проверка содержимого DMG не выполняется (скрипт не монтирует образ), только читает метаданные.
